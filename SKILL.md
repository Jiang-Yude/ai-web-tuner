# AI 網頁微調術（ai-web-tuner）

## 這個技能包在解什麼

用 AI 做網頁時，大部分內容它都做得不錯。真正卡關的，是那些「肉眼才看得出對不對」的局部細節：

- 圖片在框裡的位置偏了、頭像切到臉
- 某個元件太大或太小、邊距不對
- 配色一直配不到你要的那個調
- 滑鼠光暈、陰影太強或太淡
- 首頁大標在桌機好看，手機就爆成三行

這些東西有個共同點：你得「看到」才知道對不對，純文字描述不出來。如果交給 AI 反覆「改一個值、你截圖、它再看、再改」，又慢又燒 token，雙方都耗。

解法是換一種協作姿勢：**遇到這種肉眼微調，不要讓 AI 猜值，改成請它做一個瀏覽器內的「校稿器」（tuner）。你自己拉滑桿或按方向鍵，即時看效果，調到滿意，按一個鈕複製 CSS 貼回正式版。**

## 核心原則

1. **肉眼微調建工具，不要 AI 迭代。** 凡是「要看到才能判斷、會來回試」的視覺細節，AI 改值再截圖看是最貴的做法。建一次校稿器，使用者自己控制，省 3 到 20 倍 token，體驗也好。
2. **判斷臨界：來回試 2 到 3 次還沒定案，就給校稿器。** 只調一兩個值、一次就好的，AI 直接改加一次驗證即可，不用大費周章。
3. **多版型先給選擇題，再微調。** 配色、版型這種「方向都還沒定」的，先一次給六組並排讓使用者挑方向，挑中再用校稿器微調，不要一組一組猜。
4. **拉到極端值還是不對，回頭檢查素材。** 例如圖片怎麼調都沒有臉，多半是原圖本身就沒拍到，CSS 救不回來，先換素材。

## 何時用、何時不用

| 適合 | 不適合 |
|------|--------|
| 圖片位置／裁切、元件大小、邊距 | 整頁版型重排（直接改結構） |
| 配色、光暈、陰影、accent 色 | 文案、內容、邏輯（直接編輯） |
| 標題字級、響應式字級（桌機⇄手機） | 一次就能定的小改（直接改即可） |

## 四支校稿器（tuners/）

打開即用，也可以當範本請 AI 照著你的元件改。

| 檔案 | 解什麼 | 招牌功能 |
|------|--------|----------|
| `tuners/palette-tuner.html` | 配色配不到、一直猜 | 六組方向並排挑 → 挑中 HSL 微調 → 複製 CSS 變數 |
| `tuners/position-tuner.html` | 圖片位置、元件大小裁切 | 雙預覽（實際＋放大）＋ X/Y/縮放滑桿 ＋ 方向鍵細調 ＋ 複製 |
| `tuners/type-scale-tuner.html` | 標題字太大太小、雙語站爆行 | 桌機⇄手機即時預覽 ＋ 兩級字級 ＋ 複製 clamp() |
| `tuners/color-glow-tuner.html` | 滑鼠光暈、陰影、accent 色 | HSL 三軸 ＋ 預設色卡 ＋ 透明度/半徑 ＋ 複製 |

入口頁 `tuners/index.html`，四支都連在裡面。

## 標準流程（請 AI 照這個做）

1. **判斷**：這是「肉眼微調、會來回試」的細節嗎？是 → 進校稿器；不是 → 直接改。
2. **選對的校稿器**：位置／配色／字級／光暈，從上表挑一支。
3. **填現況**：把校稿器的預設值設成「網站現在的樣子」，使用者一打開看到的就是現狀，不是空白。
4. **使用者拉到滿意 → 複製 CSS**。
5. **貼回正式版**：精準替換對應元件那行，不要動到別的。
6. **驗證一次**：套用後看一眼，對了就收工。

## 為什麼省 token（實測對照）

以「一張圖片在五個容器、桌機手機兩種、收起展開兩態，共二十個狀態要對位」為例：

| 做法 | 估計 token |
|------|-----------|
| AI 迭代（改值→使用者截圖→AI 看→再改），每個狀態 5 輪以上 | 每張 40 至 60K |
| 建一次校稿器（瀏覽器內鍵盤調＋複製 CSS 貼回） | 2 至 3K |

省 10 到 20 倍。狀態越多、來回越久，校稿器越划算。

## 觸發詞

位置調不好、圖片偏掉、頭像切到臉、元件太大太小、裁切不對、配色一直猜、調色盤、給我色盤、光暈太強太淡、調陰影、標題字太大太小、調標題字級、雙語標題爆行、局部校稿、微調這個、給我一個可以調的工具。
